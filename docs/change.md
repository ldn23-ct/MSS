# 第二轮项目构建变更说明（修订版）

## 1. 文档目的

本文档用于说明车辆侧向背散射成像仿真项目进入第二轮构建时的背景、需求变化和新建设目标。

第一轮项目已经完成基础 Geant4 背散射仿真链路，形成了可运行的事件级 Monte Carlo 数据生成基础。第二轮项目不直接延续第一轮规格作为硬性约束，而是在第一轮实现经验的基础上，重新围绕以下两条需求线进行需求重构：

- 车辆侧向背散射 ROI 成像摸底；
- 车辆侧向背散射场景下的多重散射性质分析。

本文档不定义完整技术规格，也不直接约束全部代码实现细节。它的主要作用是：

- 说明为什么需要第二轮构建；
- 明确旧项目内容在第二轮中的参考角色；
- 概括第二轮的新研究对象和需求边界；
- 修正第二轮扫描组织方式；
- 为后续 `spec.md`、`decisions.md`、`architecture.md`、`milestones.md` 和 `acceptance_checklist.md` 提供变更背景。

---

## 2. 第一轮项目背景

第一轮项目的主要目标是建立一个可运行的 Geant4 gamma 背散射仿真程序，用于在固定几何下生成事件级 CSV 数据。

第一轮的核心关注点包括：

- 构建基础背散射几何；
- 实现源、准直器、探测面和简单模体；
- 统计到达探测器的粒子；
- 记录事件级散射信息；
- 支持单线程 / 多线程输出；
- 验证 CSV 输出链路；
- 为后续多重散射统计提供数据基础。

第一轮项目的价值在于验证了：

- Geant4 项目结构可以稳定构建；
- 事件级数据输出链路可行；
- 背散射场景下记录探测粒子信息是可实现的；
- 通过 CSV 作为仿真程序与后处理之间的数据边界是合理的；
- 分阶段构建文档能够有效约束 Codex 或其他代码生成流程。

---

## 3. 第二轮构建的触发原因

随着项目目标从简单模体背散射统计转向车辆侧向背散射 ROI 成像，第一轮需求已经不能完整覆盖当前问题。

主要变化包括：

1. **研究对象变化**  
   第一轮主要面向基础模体和背散射链路验证；第二轮面向车辆侧向 ROI 模型。

2. **几何对象变化**  
   第二轮不追求整车 CAD 复现，而是构建一个适合侧向背散射系统分析的车辆局部 ROI 几何模型。

3. **扫描方式变化**  
   第二轮改为“车辆 ROI 固定、成像头移动”的离散扫描模式。

4. **成像头设计变化**  
   成像头由射线源、狭缝准直器和虚拟探测器平面构成。射线源采用斜入射有限焦点笔形束；准直器由外部 CSV profile 定义；探测器采用单个虚拟探测平面。

5. **统计目标变化**  
   第二轮不仅需要统计探测计数，还需要讨论哪些探测粒子事件级量能够同时满足项目摸底和论文分析需求。

6. **输出数据需求变化**  
   第一轮已有字段只能作为参考，第二轮需要重新定义车辆 ROI 场景下的事件级、位姿级和扫描级统计量。

7. **建模边界变化**  
   第二轮需要明确哪些车辆结构是当前 ROI 成像对象，哪些结构暂不作为当前成像目标。

因此，第二轮构建不是对第一轮项目的小修小补，而是围绕新的车辆侧向背散射问题进行一次需求重构。

## 4. 第一轮文件在第二轮中的角色

第一轮已经完成的文件、代码和文档在第二轮中作为历史参考使用，而不是作为后续讨论和实现的硬性规格。

其参考价值包括：

- 了解第一轮项目如何组织 Geant4 代码结构；
- 参考已有事件级 CSV 输出链路；
- 参考多线程临时文件与合并策略；
- 参考宏命令、运行配置和验收文档的组织方式；
- 参考第一轮文档分层方式，例如 `spec.md`、`architecture.md`、`milestones.md`、`decisions.md`、`acceptance_checklist.md`。

但第二轮不默认继承以下内容：

- 不默认继承第一轮 CSV 字段作为最终字段；
- 不默认继承第一轮只统计简单 PMMA 模体内散射的逻辑；
- 不默认继承第一轮几何对象；
- 不默认继承第一轮探测统计量定义；
- 不默认继承镜像准直器和镜像探测器设计；
- 不默认把第一轮规格作为第二轮实现边界。

第二轮应以新需求文档为主，第一轮内容仅用于降低重新设计成本。

### 4.1 当前仓库代码现状

当前仓库仍保留第一轮 PMMA 背散射代码。该代码可作为项目结构、Geant4 action 链路、spectrum sampling、CSV 输出链路和多线程临时文件合并策略的参考。

但当前代码中的 PMMA 模体、air defect、固定三块 jaw、mirror collimator、mirror detector、macro 主配置入口、旧 compact/debug CSV schema、`hits_profile_*` 输出命名和只统计 `PMMALogical` 内散射的逻辑，均不属于第二轮实现边界。后续实现应以 `docs/spec.md` 为准进行重命名、拆分或替换。

## 5. 第二轮项目的两条需求线

第二轮项目同时服务两条需求线。

### 5.1 A 线：项目摸底需求

A 线关注系统层面的可行性摸底。

目标是模拟：

```text
斜入射有限焦点笔形束
+ 狭缝准直
+ 虚拟探测器平面
+ 车辆侧向 ROI 模型
```

在车辆侧向 ROI 场景下的成像统计表现。

A 线关注的问题包括：

- 移动成像头对固定车辆侧向局部区域是否能形成有效背散射响应；
- 不同成像头扫描位姿下探测计数如何变化；
- 车门、车窗、B 柱、座椅、内部目标物等结构是否会产生可区分的统计响应；
- 探测计数、能量分布、探测面空间分布等量能否用于形成初步成像指标；
- 哪些车辆区域适合当前侧向背散射系统探测，哪些区域不适合或只能作为遮挡区对照。

A 线不要求直接完成工程级图像重建。其重点是建立可解释、可比较的统计响应。

### 5.2 B 线：论文数据需求

B 线关注物理统计和论文分析。

目标是研究：

```text
车辆侧向背散射成像系统下，被探测粒子的多重散射性质
```

B 线关注的问题包括：

- 被探测粒子的散射次数分布；
- 单次散射与多重散射比例；
- Compton / Rayleigh 等过程的贡献；
- 探测能量与散射次数之间的关系；
- first scatter / last scatter 空间位置分布；
- 不同车辆材料区对被探测粒子的贡献；
- 内部目标物是否改变多重散射统计特征。

B 线的重点不是直接生成图像，而是生成可支撑论文分析的事件级统计数据。

## 6. 第二轮车辆 ROI 模型定位

第二轮车辆模型定义为：

```text
车辆侧向背散射 ROI 模型
```

该模型不用于整车完整扫描，也不追求真实 CAD 还原。

模型只表达侧向背散射系统可有效照射和接收的车辆侧面区域。

### 6.1 重点建模区域

车辆 ROI 模型重点包括：

- 车门；
- 车窗；
- B 柱；
- 乘员舱空气；
- 座椅 / 内饰；
- 可选内部目标物。

这些结构用于表达车辆侧面局部区域的典型遮挡、散射和空腔特征。

### 6.2 暂不作为主要成像目标的区域

以下结构不作为第二轮主要成像目标：

- 底盘；
- 发动机舱；
- 远侧车体；
- 大量金属机械结构。

这些结构必要时可作为遮挡区、无效区或不可用区域对照，但不进入当前 ROI 模型的核心表达。

### 6.3 建模原则

车辆 ROI 模型应使用 Geant4 基本几何体实现。

优先目标为：

- 几何稳定；
- 材料分区清楚；
- 可视化可检查；
- 易于修改；
- 便于固定车辆 ROI；
- 便于后续成像头扫描统计；
- 便于记录散射发生的 volume / material / region。

暂不追求：

- 曲面真实车身；
- CAD 级车辆细节；
- 复杂机械结构；
- 完整车体遮挡链路；
- 真实制造结构复现。

## 7. 第二轮扫描组织原则

第二轮采用以下扫描组织方式：

```text
固定车辆 ROI 模型
+ 移动成像头组件
+ 成像头 x-y 离散扫描位姿
+ 每个位姿独立运行一组 Monte Carlo 统计
```

### 7.1 坐标系统一

项目坐标系在整个第二轮中保持统一。

车辆 ROI 模型固定在统一 global 坐标系中，不通过整体平移表达扫描过程。

扫描过程通过改变成像头组件在统一坐标系中的离散位姿表达。

### 7.2 车辆 ROI 固定

车辆 ROI 模型在所有扫描位姿中保持固定。

固定对象包括：

- 车门；
- 车窗；
- B 柱；
- 乘员舱空气；
- 座椅 / 内饰；
- 内部目标物；
- 其他用于表达遮挡或对照的车辆结构。

车辆 ROI 固定的目的包括：

- 保持车辆结构区坐标稳定；
- 降低多组件整体平移的实现复杂度；
- 便于记录 first scatter / last scatter 所在 volume 或 material；
- 便于将扫描变量集中到成像头模块中。

### 7.3 成像头移动

成像头作为一个整体组件组进行 `x-y` 离散平移。

成像头组件包括：

- 射线源；
- 狭缝准直器；
- 虚拟探测器平面；
- 与探测器相关的接受边界和探测判定逻辑。

每一个扫描状态定义为一个成像头位姿：

```text
pose_id
head_offset_x
head_offset_y
```

成像头内部各组件在同一个 pose 内保持相对位置不变。

### 7.4 零位姿 global 坐标 + offset

第二轮成像头组件采用：

```text
零位姿 global 坐标 + 位姿 offset
```

即各组件首先在 `head_offset_x = 0`、`head_offset_y = 0` 的零位姿下给出 global 坐标。扫描时，对成像头内所有组件统一施加同一个位姿平移：

```text
x_actual = x_zero + head_offset_x
y_actual = y_zero + head_offset_y
z_actual = z_zero
```

第二轮暂不考虑：

- 成像头旋转；
- 成像头 z 方向位移；
- 连续运动；
- 运动模糊；
- 时间相关探测器积分；
- 真实机械运动控制。

### 7.5 每个位姿独立运行

每个成像头位姿作为一个静态几何状态独立运行一组 Monte Carlo 统计。

第二轮暂不要求：

- 在一个 Geant4 run 内自动完成全部位姿扫描；
- 模拟连续运动；
- 模拟运动模糊；
- 模拟时间相关探测器积分；
- 真实机械扫描过程。

## 8. 第二轮成像头设计方向

第二轮成像头由三部分构成：

```text
射线源
+ 狭缝准直器
+ 虚拟探测器平面
```

### 8.1 射线源

射线源采用斜入射有限焦点笔形束。

基本约束为：

- 每个 event 产生一个 primary gamma；
- primary gamma 起点在圆形焦点面内均匀采样；
- 圆形焦点面垂直于入射方向；
- 焦点直径默认参考值为 `5 mm`；
- primary gamma 方向固定，不再使用目标平面采样形成锥束；
- 入射方向位于 `x-z` 平面内；
- 入射方向与 `+x` 轴正方向成 `theta` 角；
- `theta` 默认参考值为 `45°`；
- `theta` 后续可配置，合法范围为 `(0°, 90°]`。

方向向量定义为：

```text
incident_dir = (cos(theta), 0, sin(theta))
```

其中所有具体数值均为当前设计参考值，不作为不可修改的物理常量。

### 8.2 狭缝准直器

狭缝准直器由外部 CSV profile 定义。

基本原则为：

- CSV 表头沿用第一版：`profile_id,jaw_id,vertex_id,x_mm,z_mm`；
- `x_mm` 和 `z_mm` 表示成像头零位姿下的 global `x-z` 坐标；
- 每个 profile 包含可变数量的钨板 jaw；
- 若某个 profile 包含 `M` 块 jaw，则 `jaw_id` 必须连续编号为 `jaw_0 ... jaw_{M-1}`；
- 每块 jaw 是零位姿 global `x-z` 平面内的凸多边形；
- 每块 jaw 顶点数 `N >= 3`；
- 每块 jaw 使用 `G4ExtrudedSolid` 沿 global `y` 方向拉伸；
- 拉伸长度当前参考值为 `120 mm`；
- 同一次 run 只读取一个 profile；
- 几何在 run 内保持不变。

第二轮不构建镜像准直器。

取消镜像的原因是斜入射破坏了第一轮中关于 `x = 0` 的左右对称假设。

### 8.3 虚拟探测器平面

探测器采用单个理想虚拟平面，不模拟真实探测器材料响应。

基本原则为：

- 探测器平面平行于 global `x-o-y` 平面；
- 探测器首先以零位姿 global 坐标给出；
- 扫描时，探测器的 `x/y` 接收范围随成像头整体平移；
- 探测器 `z` 位置当前待定；
- 探测器接收范围当前待定；
- 探测器几何位置需要与狭缝准直系统协同设计。

第二轮不构建镜像探测器。

## 9. 第二轮统计量设计方向

第二轮的核心讨论对象是：

```text
探测到的粒子应该记录哪些量，才能同时满足项目摸底和论文需求
```

统计量应分为三个层次。

### 9.1 事件级原始量

事件级原始量描述每一个被探测粒子的历史和探测状态。

候选类别包括：

- 初始能量；
- 初始位置；
- 初始方向；
- 入射角 `theta`；
- 成像头位姿 `pose_id`；
- `head_offset_x`；
- `head_offset_y`；
- 探测位置；
- 探测能量；
- 探测方向；
- 散射次数；
- 不同物理过程次数；
- first scatter 位置；
- last scatter 位置；
- first / last scatter 所在车辆结构区；
- 不同车辆材料区内的散射次数；
- 是否与内部目标物发生过相互作用。

事件级数据是后续所有位姿级和扫描级统计的基础。本文档中的事件级候选类别用于说明需求来源；第二轮本轮 Geant4 基础程序的正式 CSV / debug CSV 字段边界以 `docs/spec.md` 为准，不在本轮 CSV 中输出 `pose_id`、初始源状态、`target_interaction` 或 per-region scatter counts。

### 9.2 位姿级统计量

位姿级统计量描述某一个成像头扫描位姿下的整体响应。本节只作为下一轮后处理项目迭代的需求背景，不属于第二轮本轮 Geant4 基础程序的实现范围。

候选类别包括：

- 入射粒子数；
- 探测粒子数；
- 探测效率；
- 探测能量分布；
- 探测面空间分布；
- 单次散射比例；
- 多重散射比例；
- 平均散射次数；
- 目标物相关事件比例；
- 各车辆结构区贡献比例。

位姿级统计量主要服务项目摸底，也可作为论文中的比较单位。本轮不输出 pose-level summary、pose-level image 或位姿级统计文件。

### 9.3 扫描级统计量

扫描级统计量描述成像头在 `head_offset_x - head_offset_y` 平移网格上的整体响应。本节只作为下一轮后处理项目迭代的需求背景，不属于第二轮本轮 Geant4 基础程序的实现范围。

候选结果包括：

- 总探测效率二维图；
- 能量窗口计数二维图；
- 单次散射事件二维图；
- 多重散射事件二维图；
- 目标物相关响应图；
- 有 / 无目标物差分响应图；
- 不同结构区贡献图。

扫描级统计量用于判断当前系统是否具备形成侧向成像响应的潜力。本轮不输出 scan-level summary、二维响应图、差异图、统计指标或后处理脚本。

## 11. 第二轮当前非目标

为避免第二轮范围过大，以下内容暂不作为当前构建目标：

- 整车 CAD 复现；
- 完整车辆扫描；
- 发动机舱详细建模；
- 底盘详细建模；
- 远侧车体完整遮挡建模；
- 大量金属机械结构建模；
- 真实探测器材料响应；
- 工程级图像重建；
- 连续运动扫描；
- 运动模糊；
- 自动完成大规模扫描 campaign 的任务管理系统；
- 成像头旋转；
- 成像头 z 方向运动；
- 镜像准直器；
- 镜像探测器。

这些内容可作为后续阶段扩展，但不应进入第二轮基础构建文档的默认范围。

## 12. 变更结论

第二轮项目构建的核心变化可以概括为：

```text
从基础背散射事件级链路验证
转向固定车辆 ROI、移动成像头条件下的车辆侧向背散射统计量设计与数据生成
```

第二轮不以整车真实复现为目标，而是以稳定、可解释、可扫描的车辆侧向 ROI 模型为基础，服务两个目标：

- 项目摸底：判断移动成像头对固定车辆 ROI 是否能形成有效侧向背散射响应；
- 论文数据：分析被探测粒子的多重散射性质及其与车辆结构、能量、成像头位姿和探测响应之间的关系。

后续构建应围绕这两个目标重新定义规格、架构、里程碑和验收标准。
